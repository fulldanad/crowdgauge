Notes on Priorities

Users will rate each priority listed based on how much they value it. Each priority is represented by a circle and should have an icon assigned.

They can then see how different mechanisms will impact their priorities and read text about how each priority is affected by each mechanism.

Notes on Actions

Each general mechanism can have a number of specific actions. For example a mechanism 'Build more roads' could have these actions assigned:

  • Build 9 miles of two-lane road
  • Build 9 miles of 4-lane highway with 3 interchanges

When playing the game, users are given a fixed number of coins and asked to choose how they would allocate them. Each action is assigned a certain number of coins to indicate relative expense. Some may cost nothing.

Each column represents a particular number of coins. Add new action columns to represent a different number of coins.

In addition to cost, each action is given a multiplier to indicate how effective it is in achieving the goals of the mechanism. This multiplier is applied to the scores for each priority (both positive and negative).
